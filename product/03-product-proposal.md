## 1. 项目背景

### 1.1 AI 信息爆炸的时代背景

自 2022 年末生成式 AI 浪潮爆发以来，全球人工智能领域进入前所未有的高速迭代周期。大模型、Agent 框架、推理优化、多模态、具身智能等子方向齐头并进，每天都有大量论文、开源项目、产品发布和行业观点涌现。信息源头分散在 GitHub、HackerNews、Reddit、个人博客、厂商官方博客、国内科技媒体、Telegram 频道等十余类平台，且语种横跨中英文。

一个典型 AI 从业者每天面对的信息消费困境如下：

- **信源碎片化**：仅在 GitHub 一个平台上，karpathy 的个人动态、vllm/sglang/transformers/ollama/langchain 的 releases、Trending 列表就分散在多个页面，需要逐一访问。
- **语言壁垒**：高质量一手信息多为英文，国内从业者阅读存在理解成本，尤其涉及技术细节时。
- **噪声高**：Reddit、HackerNews 等社区信息量极大，但与 AI 前沿真正强相关的内容占比不足 10%，人工筛选耗时巨大。
- **时效性焦虑**：AI 领域 24 小时内就可能发生范式级变化（如新的推理框架发布、新的 Benchmark 刷新），错过一天的资讯可能意味着技术选型落后。

### 1.2 中文 AI 资讯市场的需求

国内 AI 从业者、研究者、技术决策者对高质量中文 AI 资讯存在持续且未被充分满足的需求。现有信息消费方式存在以下痛点：

1. **人工翻译滞后**：英文一手资讯到中文社区的传播通常存在 1-3 天延迟，且翻译质量参差不齐。
2. **聚合产品同质化**：市面上的 AI 资讯产品多为"搬运式"聚合，缺乏深度加工和价值判断，信息密度低。
3. **缺乏结构化**：缺乏按重要性排序、带背景补充和摘要的"一句话读懂"式简报。
4. **定制化能力弱**：用户无法方便地根据自身关注方向（如推理优化、多模态、Agent）调整信息源。

### 1.3 现有产品不足

| 产品类型 | 代表 | 不足 |
|---|---|---|
| RSS 阅读器 | Inoreader、Feedly | 需自行配置信源，无 AI 筛选，无中文深度加工 |
| AI 资讯公众号 | 各类"AI 日报" | 人工编辑，覆盖面有限，更新频率受限于人力 |
| 英文聚合站 | TechCrunch AI、The Rundown | 面向英文用户，无中文版，国内信源缺失 |
| 国内科技媒体 | 量子位、机器之心 | 内容深度好但以单篇深度报道为主，缺乏横向聚合简报 |
| Twitter/微博 信息流 | 社交平台 | 噪声极大，算法推荐导向不聚焦 |

市场空白明确：**一个开源、AI 驱动、中文输出、多源聚合、自动发布的每日 AI 前沿简报系统**。

## 2. 产品定位

### 2.1 一句话定位

> **AIHotNews 是一个 AI 驱动的多源新闻聚合系统，每日自动从 12 类中英文信息源采集、AI 评分筛选，并由 DeepSeek 大模型生成高质量中文 AI 前沿资讯简报，自动发布至 GitHub Pages。**

### 2.2 详细定位说明

- **目标形态**：每日一份的"AI 前沿日报"，类似科技领域的"早报"但完全由 AI 自动化生成。
- **核心技术差异化**：并非简单的 RSS 聚合，而是"采集 → AI 评分 → 背景补充 → 中文简报生成 → 自动发布"的端到端自动化流水线，AI 介入深度贯穿全链路。
- **开源可定制**：用户可 fork 仓库，自定义信息源、评分阈值、输出语言、发布渠道。
- **零成本运维**：基于 GitHub Actions + GitHub Pages，无需服务器，单次运行 API 成本约 0.1-0.5 元。
- **品牌定位**：技术驱动的"AI for AI"信息处理工具，而非人工编辑的资讯媒体。

## 4. 核心价值主张

### 4.1 六大核心价值

#### 价值一：AI 自动评分筛选

通过自研评分体系（0-10 分，重要资讯阈值 8.0），AI 对每条采集内容进行重要性打分，过滤低价值噪声，仅保留真正值得关注的前沿资讯。实测数据：24 小时窗口内可从约 35 条采集内容中筛选出 8 条重要资讯，信息密度提升 4 倍以上。

#### 价值二：中文深度简报

由 DeepSeek deepseek-chat 模型（温度 0.3、max_tokens 8192）生成结构化中文简报，每条资讯包含：标题、核心内容、背景补充、影响分析、原始链接。不是简单翻译，而是基于多源信息的二次加工和价值提炼。

#### 价值三：多源聚合

覆盖 12 类中英文信息源：

- **GitHub 动态**：karpathy 个人动态，vllm/sglang/transformers/ollama/langchain 项目 releases
- **HackerNews**：Top 40 stories，最低分数 100
- **Reddit**：MachineLearning / LocalLLaMA / artificial / singularity 四个子版块
- **国际 RSS**：Simon Willison 博客、GitHub Trending、Semianalysis、OpenAI Blog、Anthropic News、Google AI Blog、Hugging Face Blog、LWN.net
- **国内中文源**：量子位、新智元、36氪快讯、机器之心、虎嗅
- **Telegram**：zaihuapd 频道

横跨开发者社区、厂商官方、独立博客、国内媒体、即时通讯五大信息生态，覆盖广度行业领先。

#### 价值四：开源可定制

完整代码开源，用户可自行 fork 仓库，调整：
- 信息源增删（修改 scrapers 配置）
- 评分阈值（调整 analyzer 参数）
- 输出语言（修改 summarizer prompt）
- 发布渠道（替换 webhook / Pages 配置）
- 大模型 Provider（替换为其他 OpenAI 兼容 API）

#### 价值五：自动发布

GitHub Actions 定时任务（UTC 22:00 每日执行）自动完成"采集 → 分析 → 生成 → 发布"全流程，GitHub Pages + Jekyll 免费托管，玻璃拟态 UI 设计。用户只需订阅，无需任何手动操作。

#### 价值六：零成本运维

- **基础设施成本**：GitHub Actions（免费额度）+ GitHub Pages（免费托管）= 0 元
- **API 成本**：DeepSeek API 约 0.1-0.5 元/天
- **月度总成本**：约 3-15 元
- **运维人力**：0（全自动化）

## 6. 商业模式

### 6.1 核心原则：开源免费

产品核心功能永久开源免费，包括采集、分析、生成、发布全链路。社区可自由 fork、修改、部署。

### 6.2 增值服务（规划中）

| 服务 | 定价模式 | 目标客户 | 价值 |
|---|---|---|---|
| **私有部署咨询** | 按项目收费 | 企业 AI 团队 | 帮助企业搭建内部 AI 资讯系统，定制信息源 |
| **定制信息源** | 按月订阅 | 垂直领域团队 | 针对特定方向（如具身智能、芯片）定制采集源 |
| **企业版** | 按席位/年 | 中大型企业 | 团队订阅、管理后台、合规审计、SSO |
| **API 服务** | 按调用计费 | 开发者 | 提供结构化资讯 API，支持二次开发 |
| **高级简报** | 按月订阅 | 决策者 | 周报、月报、深度专题、行业图谱 |

### 6.3 收入预期

- **首年**：以免费产品积累用户，增值服务收入预期较低（< 10 万元）
- **第二年**：企业版与 API 服务上线，预期年收入 50-100 万元
- **第三年**：形成稳定订阅收入，预期年收入 200-500 万元

## 9. 风险评估

### 9.1 技术风险

| 风险 | 等级 | 影响 | 应对策略 |
|---|---|---|---|
| DeepSeek API 不可用或涨价 | 中 | 简报生成中断 | 支持多 Provider（OpenAI、通义千问、Moonshot），抽象 LLM 接口层 |
| 信息源反爬升级 | 中 | 采集成功率下降 | 每源独立 scraper，单源失败不影响整体；监控采集成功率 |
| GitHub Actions 限额 | 低 | 自动化中断 | 控制单次运行时长 < 30 分钟；必要时迁移至自建 Cron |
| 大模型输出质量波动 | 中 | 简报质量下降 | 温度参数固定 0.3，建立质量抽检机制 |

### 9.2 运营风险

| 风险 | 等级 | 影响 | 应对策略 |
|---|---|---|---|
| 用户增长缓慢 | 中 | 影响社区化和商业化 | 强化内容质量，主动推广至社区；SEO 优化 |
| 信息源时效性下降 | 低 | 内容新鲜度降低 | 定期评估信源质量，动态调整 |
| 维护精力不足 | 中 | 更新停滞 | 模块化设计，降低维护成本；接受社区贡献 |

### 9.3 合规风险

| 风险 | 等级 | 影响 | 应对策略 |
|---|---|---|---|
| 内容版权争议 | 中 | 法律风险 | 仅做摘要与链接聚合，不全文转载；标注原始来源 |
| 信息源使用条款 | 低 | 采集受限 | 遵守各平台 robots.txt 和 API 使用条款；优先使用官方 API |
| AI 生成内容标注 | 低 | 合规要求 | 明确标注"AI 生成"，区分编辑内容与自动化内容 |
| 数据隐私 | 低 | 合规风险 | 不采集个人数据，仅处理公开资讯 |
